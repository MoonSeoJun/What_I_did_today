# 1. 서버로서의 노드
## 장점

- libuv 라이브러리를 사용하여 I/O 작업을 논 블로킹 방식으로 처리하여 스레드 하나가 많은 수의 I/O를 혼자서도 감당할 수 있다
- 멀티 스레드 방식보다 쉬움
- 웹 서버가 내장되어 있음
- JSON 형식과 쉽게 호환됨

## 단점

- CPU 부하가 큰 작업에는 적합하지 않다.
- 하나 뿐인 스레드가 멈추지 않도록 관리가 필요함
- 어중간한 성능
- 서버가 커졌을 때 서버를 관리하기 어려움

## 활용 방안

> 개수는 많지만 크기는 작은 데이터를 실시간으로 주고받는 데 적합하다. 네트워크나 데이터베이스, 디스크 작업 같은 I/O에 특화되어 있기 때문이다.
